# Global Tech Stack

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global tech stack.

## When to use this skill

- When making architectural decisions about which technologies or frameworks to use
- When adding new dependencies to package files (package.json, Cargo.toml, requirements.txt)
- When integrating third-party services (authentication, email, monitoring, payment processors)
- When choosing between different libraries or approaches for implementing features
- When setting up development environments or configuring build tools
- When documenting technology choices in README or architecture documents
- When reviewing the existing tech stack to understand project conventions
- When planning deployment or infrastructure changes
- When onboarding new team members and introducing them to the stack
- When ensuring consistency in tool choices across the project
